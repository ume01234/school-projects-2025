
# テーマ案：「接待AIは人間のパフォーマンスをどう変えるか」

## 1. 研究のコアアイデア

- オセロの対戦AIに「ユーザーと接戦になること」を目標とする報酬関数を与えた **接待AI（Close-Game AI）** を強化学習で構築する。
- この接待AIと、通常の「勝利最大化AI（ガチAI）」を比較し、
  - 人間の対戦パフォーマンス（勝率、明らかな悪手率、リスクテイク傾向）
  - 主観的体験（楽しさ、緊張感、モチベーション、自己効力感）
  - 連続セッションを通した上達度（簡易指標）
  を評価する。

## 2. 位置付けと先行研究との違い

- 近年、**Dynamic Difficulty Adjustment (DDA)** を強化学習やQ-learningで実装し、プレイヤーのスコアや体験を最適化する研究が多いが、対象は主に一人用ゲームやゲーム内パラメータの調整である。[web:16][web:18][web:20][web:26]
- 人間とAIのハイブリッド性能や、人間の意思決定へのAIの影響を分析する研究はあるが、AIは通常「勝つ・支援する」ことを目的としており、「接戦を演出する対戦相手」として設計された例はほとんどない。[web:19][web:22][web:28][web:30]
- 本テーマは、
  - 零和ボードゲーム（オセロ）
  - 「接戦を維持する」ことを報酬とする対戦AI
  - 人間のパフォーマンス・心理・上達への影響
  を組み合わせる点でニッチかつ新規性がある。

## 3. 接待AIの設計案

### 3.1 環境

- ゲーム：オセロ（Reversi）
- 実装：OpenSpiel 等の既存環境を利用して実装負荷を削減。[web:11]

### 3.2 ベースラインAI（ガチAI）

- 目的：勝利確率最大化。
- 手法案：
  - シンプルな強化学習（DQN / PPO 等）による方策学習
  - もしくは既存の強いルールベース／ミニマックス／AlphaZero系実装を利用

### 3.3 接待AI（Close-Game AI）

- 目標：人間とのスコア差が小さい、かつ「接戦感のある」対局を実現する。
- 報酬設計案：
  - 最終スコア差 \(|score_{AI} - score_{human}|\) を小さくするほど高報酬。
  - 終盤までゲームが長引く（早期に一方的展開にならない）ほどボーナス。
  - 必要に応じて「人間が逆転可能な局面」を作る行動にボーナスを与える。
- 拡張案：
  - Few-shot opponent modeling：数手のプレイから人間の強さ・傾向を推定し、それに応じて接戦度合いを調整する。

## 4. 実験デザイン案

### 4.1 実験条件（例：2×2 デザイン）

- 軸1：AIの種類
  - ガチAI（勝利最大化）
  - 接待AI（接戦最大化）
- 軸2：情報提示
  - 正直な優勢劣勢グラフ（評価値や石数をそのまま表示）
  - バイアス付きグラフ（やや楽観／悲観寄りの表示）

→ 計 4 条件：
1. ガチAI ＋ 正直グラフ  
2. ガチAI ＋ バイアスグラフ  
3. 接待AI ＋ 正直グラフ  
4. 接待AI ＋ バイアスグラフ  

### 4.2 測定指標

- **客観指標**
  - 勝率・最終スコア差
  - 明らかな悪手率（簡易なヒューリスティックや既存AIの評価値で定義）
  - リスクテイク傾向（形勢が悪いときに大きく形勢を変える手を選ぶ頻度など）
  - 対局時間・手番ごとの思考時間

- **主観指標（質問紙）**
  - 楽しさ・没入感
  - 緊張感・ストレス
  - 自己効力感（「自分はこのゲームが上達していると感じる」など）
  - AIへの信頼感・公平性の評価

- **上達指標（複数セッション時）**
  - 序盤の定石的な手の頻度
  - 悪手率の時間的変化
  - 同じ被験者がガチAIと対戦したときの勝率の変化

## 5. 実装方針（負荷を抑える工夫）

1. **ステップ1：環境とガチAI**
   - OpenSpielなどを用いてオセロ環境を構築。
   - 既存の強いAIを利用するか、簡易な強化学習で「そこそこ強い」ガチAIを用意。

2. **ステップ2：接待AIの学習**
   - 報酬を「スコア差の絶対値が小さいほど高い」に変更し、ガチAIと同じアルゴリズムで学習。
   - 必要なら、対戦相手の強さを固定した簡易シミュレーション環境で学習し、人間実験時にはその方策を固定して使う。

3. **ステップ3：実験用UI**
   - Webベースの対戦UIを作成（AI側の計算はサーバーで実行）。
   - 優勢劣勢グラフの表示機能と、バイアスをかけるオプションを実装。
   - 対戦終了時に質問紙を表示し、データを自動保存。

4. **ステップ4：ユーザー実験・分析**
   - 少人数からパイロット実験を行い、指標や質問項目をチューニング。
   - 本実験で4条件を比較し、「接待AIが人間のパフォーマンスと体験に与える影響」を統計的に評価。

## 6. 想定される主張・貢献

- 接戦志向の対戦AI（接待AI）は、従来のDDAやガチAIと異なる特徴的な体験・行動変化を引き起こす可能性がある。
- 条件によっては、
  - 楽しさやモチベーションは向上する一方、
  - 上達や自己評価の正確性にはプラス/マイナス両面の影響がある
  などの結果が期待される。
- 零和ボードゲームにおける「接戦を演出するAI」という新しい設計パラダイムを提示し、人間‐AIインタラクション研究・DDA研究の新たなトピックになりうる。
